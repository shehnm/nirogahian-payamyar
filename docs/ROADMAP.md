## بخش الف — اهداف ریپو (= خواست کارفرما)

این ریپو برای **رسیدن به تحویل فاز ۲** است، نه فقط نگه داشتن سایت.

مرجع کامل: [CLIENT-REQUEST.md](CLIENT-REQUEST.md)

### هدف نهایی (تعریف موفقیت)

```
[کارفرما] وقتی خبر منتشر می‌کند
    -> همه دسته‌ها (~۷۰۰) در تنظیمات پیام‌یار دیده شود
    -> «اخبار» و دسته‌های لازم قابل تیک باشند
    -> ارسال خودکار به ایتا، بله، گپ، روبیکا برود
    -> بدون خطای critical
    -> سایت up بماند
```

### اولویت‌ها (وقتی تعارض داری)

| رتبه | اولویت | توضیح |
|------|--------|--------|
| ۱ | **سایت up** | هیچ پچی که کل سایت یا wp-admin را down کند قابل قبول نیست |
| ۲ | **همه دسته‌ها در UI** | نه ۳، نه ۱۰۰ — هدف ~۷۰۰ |
| ۳ | **ارسال خودکار** | publish بدون دکمه خاکستری |
| ۴ | سرعت صفحه تنظیمات | زیر ~۳۰ ثانیه قابل تحمل؛ بالاتر = شکست UX نه لزوماً rollback |

### خارج از تعهد فاز ۲

- افزونه تلگرام جدا
- لایسنس zhaket
- بهینه‌سازی کلی ۴۸هزار پست

### معیار «تمام شد» (چک قبل از اعلام به کارفرما)

از [TEST-CHECKLIST.md](TEST-CHECKLIST.md) — همه آیتم‌ها تیک بخورند.

## بخش ب — بن‌بست‌ها (تکرار نکن)

هر ردیف = راهی که رفتیم و **شکست خورد** یا **سایت را خراب کرد**.

### جدول بن‌بست‌ها

| ID | چه کردیم | علت شکست | عارضه روی سایت | وضعیت |
|----|----------|----------|----------------|--------|
| **D01** | پچ اولیه ~۴۳۸۲ — cap ساده | لیست دسته خالی ماند | پیام‌یار بی‌فایده | بن‌بست |
| **D02** | v4b/c/d — `merge` روی نتایج get_terms | تداخل با ساختار افزونه | کرش / صفحه سیاه پیام‌یار | **ممنوع** |
| **D03** | v4b/c/d — `parent=0` اجباری | دسته‌ها wrong tree | کرش یا لیست غلط | **ممنوع** |
| **D04** | v4b/c/d — `rest_api` در admin | بار اضافی / حلقه | صفحه پیام‌یار کرش | **ممنوع** |
| **D05** | فقط cache بزرگ‌تر بدون pagination | همه termها یک‌جا در حافظه/UI | صفحه خیلی کند، عملاً باز نشد | **ممنوع** |
| **D06** | STABLE-4624 به‌عنوان «راه‌حل نهایی» | `HARD_CUTOFF` + `number=100` + قطع بعد از ۳ query | فقط ~۳ دسته؛ ارسال خودکار نشکست | **موقت OK — هدف نیست** |
| **D07** | v5 + v4 **همزمان** فعال | `py_log()` دوبار define | **Fatal — کل admin** | **ممنوع** |
| **D08** | v5 — `terms_pre_query` + `get_terms()` داخل loader | recursion / بار سنگین بدون `pad_counts` | critical error پیام‌یار | **ممنوع** |
| **D09** | v5b/v5c روی **پروداکشن** بدون استیجینگ | همان + احتمال load ناقص فایل | critical error | **ممنوع تا استیجینگ** |
| **D10** | paste ناقص در AXFilePro | syntax error | critical error | **ممنوع — فقط Upload** |
| **D11** | `posts_pre_query` برگرداند `array()` (HARD_CUTOFF پست) | hook انتشار خالی می‌ماند | ارسال خودکار نمی‌رود | **ممنوع در v6** |
| **D12** | deploy از IP خارج بدون File Manager | FTP timeout | اصلاً آپلود/تست نمی‌شود | محیطی — نه پچ |
| **D13** | حذف mu-plugin بدون جایگزین روی سایت پرپست | Payamyar خام = query سنگین | سایت down | **ممنوع** |

### الگوهای ممنوع (خلاصه یک خط)

```
❌ merge / parent=0 / rest_api تهاجمی
❌ دو فایل payamyar*.php فعال
❌ get_terms داخل terms_pre_query بدون guard درست
❌ cache همه دسته‌ها بدون pad_counts=false
❌ posts_pre_query => []
❌ تست پچ جدید روی پروداکشن بدون بکاپ و استیجینگ
❌ توابع global بدون پیشوند یکتا (py_v6_)
```
